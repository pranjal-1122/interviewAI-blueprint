// 01 — Local AI Model Stack

Three Models, All In-Browser

You don't download these manually. Transformers.js fetches them from Hugging Face CDN automatically the first time each pipeline runs, then caches them in the browser forever. One npm install @huggingface/transformers — that's all the setup you do.

Role Model Size Why This One
LLM / Interviewer Brain Xenova/Qwen2.5-0.5B-Instruct ~400MB Smallest capable instruction-following model. Fast inference on CPU. Good enough for interview Q&A + evaluation. Loads in ~30s on average laptop.
Speech-to-Text Xenova/whisper-small ~244MB Best accuracy/size tradeoff. Handles accents well — important for India. Chunked audio streaming means near-real-time transcription.
Text-to-Speech Xenova/speecht5_tts ~200MB Natural-sounding English voice. Runs fast. Makes the AI interviewer feel real — critical for the immersive interview experience.

The One-Time Download Problem — Solved

~850MB of models downloading on first visit would break the demo. The fix: build a pre-load screen with individual progress bars for each model. Show it as a feature — "Setting up your personal AI interview panel..." with three animated progress bars. The night before your hackathon demo, open the app once and let everything cache. From that point, startup is instant, works completely offline, and never needs the internet again. This becomes a story: "Your interview data never leaves your device."

// How model loading actually works — Transformers.js handles everything import { pipeline, env } from '@huggingface/transformers'; // Use WebGPU if available for faster inference, fallback to WASM env.backends.onnx.wasm.proxy = true; // Models auto-download from HF CDN, cache in browser IndexedDB const transcriber = await pipeline('automatic-speech-recognition', 'Xenova/whisper-small'); const interviewer = await pipeline('text-generation', 'Xenova/Qwen2.5-0.5B-Instruct'); const speaker = await pipeline('text-to-speech', 'Xenova/speecht5_tts'); // That's it. // 02 — AI Strategy: RAG + Prompt Engineering

Why This Beats Fine-Tuning

Fine-tuning requires thousands of labeled examples, GPU compute, and days of time. RAG + prompt engineering gives you the same behavioral customization in hours — and it's dynamic, adapting to every candidate in real time. This is the mature engineering decision.

What RAG Does Here

You build a static JSON knowledge base of interview questions, ideal answer frameworks, role-specific topic trees, and competency definitions. At runtime, the user's role description is used to retrieve the most relevant context and inject it into the model prompt — making a generic 0.5B model behave like a domain expert.

No Training Required

What Prompt Engineering Does

A precisely crafted system prompt shapes the model's persona, tone, question style, and evaluation rubric. It tells the model exactly what to do with weak answers (probe deeper) vs strong answers (escalate difficulty) — replicating fine-tuned behavior without any training.

Replaces Fine-Tuning

RAG Pipeline Flow

01
User Describes Target Role
e.g. "Frontend Engineer at a product startup, 2 years experience, React + TypeScript"
02
Role DNA Parser Extracts Signals
Parses role, seniority, domain, and company type. Maps to a topic tree: [React, Performance, System Design, Team Collaboration, Product Thinking]
03
RAG Retrieves Relevant Context
From your local JSON knowledge base: pulls top 5 questions per topic, STAR answer examples for that domain, startup culture signals, difficulty calibration for the seniority level
04
Context Injected into System Prompt
The Qwen model receives a rich, role-specific context window. It now "knows" this domain — without any training, purely from prompt injection.
05
Adaptive Loop — Every Answer Updates Context
After each answer, detected weak areas are appended to the live context. The next question targets that weakness specifically. This is the adaptive intelligence loop — running entirely local.
// System prompt structure — shapes model behavior without fine-tuning const buildPrompt = (role, retrievedContext, answerHistory) => ` You are Alex, a Senior Engineering Manager conducting a real job interview. ROLE CONTEXT (retrieved): ${retrievedContext.topicTree} ${retrievedContext.questionBank} ${retrievedContext.idealAnswerExamples} BEHAVIORAL RULES: - Ask ONE question at a time. Never two. - Keep questions under 25 words. - If the answer scores below 6/10 → probe the same topic deeper. - If the answer scores above 8/10 → escalate difficulty immediately. - Never reveal you are an AI model. - Use natural, conversational language. DETECTED WEAK AREAS SO FAR: ${answerHistory.weakAreas.join(', ')} QUESTIONS ASKED: ${answerHistory.questions.length}/8 Generate the next interview question only. No preamble. `;

Dev 2 — AI Pipeline

Hour 0–4

Install @huggingface/transformers. Test all three pipelines (STT, LLM, TTS) in isolation. Confirm WebGPU availability. Measure load times and inference speed.
Hour 4–10
RAG Knowledge Base + Prompt Engineering
Build the JSON knowledge base: 5 role types × 6 topics × 5 questions = 150 questions. Write and test all system prompts for each persona. Validate adaptive behavior.
Hour 10–18
STT Pipeline + Turn Detection

Hour 28–36
Integration QA + Performance
